vue2之路由跳转

步骤

①、安装路由

npm install vue-router

②、可以在package.json文件中看到已安装的路由

③、新建一个router文件夹，创建index.js文件用来存放路由

//引入vue实例
import Vue from 'vue'
//引入路由实例
import VueRouter from 'vue-router'
//引入所需要的组件
import Home from '../components/Home.vue'
import About from '../components/About.vue'
import HelloWorld from '../components/HelloWorld.vue'

//这一步非常重要，路由实例需要被vue实例使用
Vue.use(VueRouter)

const routes = [
   {
        path: '/',
        component: Home
    },
    {
        path: '/about',
        component: About
    },
    {
        path: '/helloworld',
        component: HelloWorld
    }
]

//这里是设置一些路由模式，以及绑定路由
const router = new VueRouter({
  mode: 'hash',
  base: process.env.BASE_URL,
  routes
})

//将路由暴露出去在main.js中使用
export default router

④、将路由在程序入口文件main.js中引入

import Vue from 'vue'
import router from './router'

new Vue({
  //将路由挂载到vue实例上
  router,
  render: h => h(App)
}).$mount('#app')

动态路由匹配(参数携带)

在路由跳转时携带参数，并获取到相应的参数

例如

<router-link to="/User/12/swx">12用户</router-link> |
 <router-link to="/User/13/pyq">13用户</router-link>
{
    	//获取路由跳转时携带的参数
        path: '/User/:id/:username',
        component: User
}

在user组件中显示参数

<template>
    <p>my id is {{ $route.params.id }}</p>
    <p>my username is {{ $route.params.username }}</p>
</template>

说明：

一个“路径参数”使用冒号 : 标记。当匹配到一个路由时，参数值会被设置到 this.$route.params，可以在每个组件内使用。于是，我们可以更新 User 的模板，输出当前用户的 ID：

除了 $route.params 外，$route 对象还提供了其它有用的信息，例如，$route.query (如果 URL 中有查询参数)、$route.hash 等等。你可以查看 API 文档 的详细说明。

路由匹配语法(参数约束)

在参数中自定义正则对参数的类型进行约束

(\\d+)：约束参数类型只能是数字类型

{
    //获取路由跳转时携带的参数
    path: '/User/:id(\\d+)',
    component: User
}
<router-link to="/User/12">12用户</router-link> |
//传一个字符串tom是获取不了路由参数的，因为约束了只能传数字类型
<router-link to="/User/tom">13用户</router-link>

可重复参数

**(\\d+)：参数可以是一个数组*

{
    //获取路由跳转时携带的参数
    path: '/User/:id(\\d+)*',
    component: User
}
<router-link to="/User/12/23/15/24">12用户</router-link> |
<template>
    //my id is [ "12", "23", "15", "24" ]
    <p>my id is {{ $route.params.id }}</p>
</template>

可选参数

?：意思就是有无这个参数都可以匹配到路由

{
        //获取路由跳转时携带的参数
        path: '/User/:id?',
        component: User
    }
<router-link to="/User/12">12用户</router-link> |
//虽然没有携带参数，但是它依然可以被匹配到路由
<router-link to="/User">13用户</router-link>

命名路由

有时候，通过一个名称来标识一个路由显得更方便一些，特别是在链接一个路由，或者是执行一些跳转的时候。你可以在创建 Router 实例的时候，在 routes 配置中给某个路由设置名称。

给路由组件命名

import {createRouter,createWebHashHistory} from 'vue-router'
import Home from '../components/Home.vue'
import About from '../components/About.vue'
import User from '../components/user/Index.vue'
import Profile from '../components/user/Profile.vue'
import Posts from '../components/user/Posts.vue'

const routes = [
    {
        path: '/',
        name: 'home',
        component: Home
    },
    {
        path: '/about',
        name: 'about',
        component: About
    },
    {
        //获取路由跳转时携带的参数
        path: '/User/:id/',
        name: 'user',
        component: User,
        children: [
            {
                path: 'profile',
                name: 'profile',
                component: Profile
            },
            {
                path: 'posts',
                name: 'posts',
                component: Posts
            }
        ]
    }
]

export default createRouter({
    history: createWebHashHistory(),
    routes
})

App.vue组件

<template>
  <img alt="Vue logo" src="./assets/logo.png">
  <h1>Hello App!</h1>
        <p>
            <!-- 使用 router-link 组件来导航. -->
            <!-- 通过传入 `to` 属性指定链接. -->
            <!-- <router-link> 默认会被渲染成一个 `<a>` 标签 -->
            <router-link :to="{name: 'home'}">Go to Home</router-link> |
            <router-link :to="{name: 'about'}">Go to About</router-link> |
            <router-link :to="{name: 'user',params:{ id:12 }}">12 用户</router-link> |
            <router-link :to="{name: 'user',params:{ id:13 }}">13 用户</router-link> |
        </p>
        <!-- 路由出口 -->
        <!-- 路由匹配到的组件将渲染在这里 -->
        <router-view></router-view>
</template>

要链接到一个命名路由，可以给 router-link 的 to 属性传一个对象：

<router-link :to="{name: 'user',params:{ id:12 }}">12 用户</router-link>

这跟代码调用 router.push() 是一回事：

router.push({ name: 'user', params: { id: 12 } })

这两种方式都会把路由导航到 /user/12 路径。

命名视图

有时候想同时 (同级) 展示多个视图，而不是嵌套展示，例如创建一个布局，有 sidebar (侧导航) 和 main (主内容) 两个视图，这个时候命名视图就派上用场了。你可以在界面中拥有多个单独命名的视图，而不是只有一个单独的出口。如果 router-view 没有设置名字，那么默认为 default。

<template>
  <img alt="Vue logo" src="./assets/logo.png">
  <h1>Hello App!</h1>
        <p>
            <!-- 使用 router-link 组件来导航. -->
            <!-- 通过传入 `to` 属性指定链接. -->
            <!-- <router-link> 默认会被渲染成一个 `<a>` 标签 -->
            <router-link :to="{name: 'home'}">Go to Home</router-link> |
            <router-link :to="{name: 'about'}">Go to About</router-link> |
            <router-link :to="{name: 'user',params:{id:12}}">12 用户</router-link> |
            <router-link :to="{name: 'user',params:{id:13}}">13 用户</router-link> |
        </p>
        <!-- 路由出口 -->
        <!-- 路由匹配到的组件将渲染在这里 -->
        <router-view></router-view>
        <!--命名路由视图-->
        <router-view name="leftRouter"></router-view>
</template>

一个视图使用一个组件渲染，因此对于同个路由，多个视图就需要多个组件。确保正确使用 components 配置 (带上 s)：

{
        path: '/',
        name: 'home',
        components: {
            default: Home,
            leftRouter: About
        }
},
{
        path: '/about',
        name: 'about',
        components: {
            default: About,
            leftRouter: About
        }
},

编程式的导航

当你点击 <router-link> 时，这个方法会在内部调用，所以说，点击 <router-link :to="..."> 等同于调用 router.push(...)。

声明式 编程式
<router-link :to="..."> router.push(...)

该方法的参数可以是一个字符串路径，或者一个描述地址的对象。例如：

// 字符串
router.push('home')

// 对象
router.push({ path: 'home' })

// 命名的路由
router.push({ name: 'user', params: { Id: '12' }})

// 带查询参数，变成 /register?plan=private
router.push({ path: 'register', query: { plan: 'private' }})

注意：如果提供了 path，params 会被忽略，上述例子中的 query 并不属于这种情况。取而代之的是下面例子的做法，你需要提供路由的 name 或手写完整的带有参数的 path：

const userId = '123'
router.push({ name: 'user', params: { userId }}) // -> /user/123
router.push({ path: `/user/${userId}` }) // -> /user/123
// 这里的 params 不生效
router.push({ path: '/user', params: { userId }}) // -> /user
<template>
        <p>
            <!-- 使用 router-link 组件来导航. -->
            <!-- 通过传入 `to` 属性指定链接. -->
            <!-- <router-link> 默认会被渲染成一个 `<a>` 标签 -->
            <router-link :to="{name: 'home'}">Go to Home</router-link> |
            <router-link :to="{name: 'about'}">Go to About</router-link> |
            <router-link :to="{name: 'user',params:{id:12}}">12 用户</router-link> |
            <router-link :to="{name: 'user',params:{id:13}}">13 用户</router-link> |
        </p>
        <!-- 路由出口 -->
        <!-- 路由匹配到的组件将渲染在这里 -->
        <router-view></router-view>

        <router-view name="leftRouter"></router-view>
</template>

<script>

export default {
  name: 'App',
  mounted(){
    setTimeout(() => {
      //编程式路由导航,3s后跳转到about组件
      this.$router.push({name: 'about'})
    }, 3000);
  }
}
</script>

HTML5 History 模式

如果是hash模式，路径会带有一个#，个人觉得不好看

如果是history模式，自然不会带#

不过这种模式要玩好，还需要后台配置支持。因为我们的应用是个单页客户端应用，如果后台没有正确的配置，当用户在浏览器直接访问 http://oursite.com/user/id 就会返回 404，这就不好看了。

解决方式就是在部署好项目后，在nginx中添加

location / {
  try_files $uri $uri/ /index.html;
}

这样就不会报404,会去找index.html

import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import { createStore } from 'vuex'

// 创建一个新的 store 实例
const store = createStore({
    //相当于一个data,但是它可以让全局组件访问
    //其实，有state属性就可以做到全局共享变量的，我们甚至可以通过state.count来修改对应的值。

  state () {

    }

  mutations: {
    increment (state) {
      state.count++
    }
  }
})

app.use(store)
app.mount('#app')

3.2、解构访问数据

疑问？为什么解构：原因是非解构访问数据麻烦：如：太长

<template>
  <div>
     <!-- 解构 -->
    <h1>{{ user.username }}</h1>
    <h1>{{ user.password }}</h1>
  </div>
</template>
  
  
<script setup>
import { userStore } from "../stores/user"

const state = userStore()
//将state解构出来
const { user } = state

</script>
  
<style lang="scss" scoped>
</style>

注意：将state解构出来const { user } = state这种是有问题的，这样后user就不是响应式的数据了

解决：解构时使用storeToRefs解构，让ref作响应式代理

<script setup>
import { storeToRefs } from "pinia"
import { userStore } from "../stores/user"

const state = userStore()
//解构时使用storeToRefs解构，让ref作响应式代理
const { user } = storeToRefs(state)
//做脚本处理时需要.value才能获取到值，因为已经被ref代理了
console.log(user.value.password)

</script>

总结：如果数据复杂就解构，解构时注意以上注意点

父传子

基本思想：

  • 父组件中给子组件绑定要传的属性
  • 子组件内部通过props选项接收

父组件

<template>
  <div>
    <h1>登录页面</h1>
    <!-- 1、绑定要传的属性（数据） -->
    <Home :message="msg" :count="count"/>
  </div>
</template>
  
<script setup>
import Home from "../views/Home.vue"
import { ref} from "vue"
    
const msg = ref("i am father")
const count = ref(100)
</script>

子组件

<template>
  <div>
    <h1>home主页</h1>
    <h2>父组件数据展示：{{ message }} --- {{ count }}</h2>
  </div>
</template> 
    
<script setup>
//2、接收父组件传过来的数据
//把父组件传过来的全部值由props接收，
//这样子组件就可以完全使用数据了
const props = defineProps({
  message: String,
  count: Number
})
//通过props.可以拿到父组件传过来的所有的数据
console.log(props.message)
</script>

子传父

基本思想

  • 父组件中给子组件标签通过@绑定事件
  • 子组件内部通过$emit方法触发事件

父组件

<template>
  <div>
    <h1>父组件</h1>
    <!-- 1、绑定自定义事件 -->
    <Home @get-message="getMessage"/>
    <h2>子组件数据展示：{{ message }}</h2>
  </div>
</template>
  
<script setup>
import Home from "../views/Home.vue"
import { ref} from "vue"

const message = ref("")
const getMessage = (msg) => {
  message.value = msg;
}
</script>

子组件

<template>
  <div>
    <h1>子组件</h1>
    <el-button type="primary" @click="sendMsg">子组件触发自定义事件</el-button>
  </div>
</template> 
    
<script setup>
//2、通过defineEmits获取emit,相当于$emit
const emit = defineEmits(['get-message'])

//触发子组件事件
const sendMsg = () => {
  //触发自定义事件，传数据给父组件
  //参数1：自定义事件名称
  //参数2：要传的数据
  emit('get-message',"我是子组件传过来的数据")
}

</script>
